UI Folder Summary (`client/ui/`)
================================

Core entrypoint / wiring
------------------------
- chat.py
  Main NiceGUI UI entrypoint. Builds login/register/loading screens, then swaps into the chat app view.
  Also wires auth success -> load chats/messages -> refresh sidebar -> show chats.

State & data
------------
- app_state.py
  Central UI state store (shared module-level dicts): selected chat, search query, loading state,
  in-memory chats list, and messages map. Includes helpers like chat lookup and load_chat_data().

- authentication.py
  Client-side auth functions (dummy for now). Returns {status, message, user_id} for login/register.

- fetch_chat_data.py
  Dummy “data layer” that returns UI-ready chat list and messages in the exact shapes the UI expects.

Auth / screens
--------------
- auth_page.py
  Shared auth page builder used by both login and register pages: gradient background, brand header,
  form card, inputs, submit button, footer link, and red error handling on failure.

- login_ui.py
  Login page wrapper built on top of auth_page.py. Wires “Register here” footer action.

- register_ui.py
  Register page wrapper built on top of auth_page.py. Wires “Login here!” footer action.

- loading_ui.py
  Loading screen UI shown while chats/messages are being fetched. Displays logo + title and rotates
  secondary messages every few seconds.

- welcome_ui.py
  Welcome/empty-state screen shown inside the chat app until a chat is selected.

Chat app UI components
----------------------
- sidebar.py
  Sidebar UI (Chats title, search input, chat list). Exposes render_chat_lists() and uses a callback
  for chat selection.

- chat_layout.py
  Orchestrates the right pane (welcome -> chat switch), header updates (avatar/title/date),
  and triggers re-renders when a chat is selected.

- messages_view.py
  Renders the messages list for the currently selected chat (and auto-scrolls to bottom safely).

- bottom_input_bar.py
  Bottom message input bar: text input + emoji picker + send button. Calls on_send(text).

Emoji support
-------------
- emoji_data.py
  Emoji lists and alias mappings used by the picker/search.

- emoji_picker.py
  (Currently unused by the latest chat flow) Older/alternative emoji picker implementation.

- category_svgs.py
  SVG icons for emoji categories used in the emoji picker UI.

Styling
-------
- styles.py
  Global CSS injection for the app + helper to set the page background image/color.

Static assets
-------------
- assets/
  Static files served at /static via NiceGUI (see chat.py app.add_static_files()).
  - email.svg: icon for the email field
  - password.svg: icon for the password field
  - gradient.svg: login/register background gradient
  - thick_logo.svg: brand icon used in auth + loading
  - wallpaper.svg: chat app background decoration
  - logo.svg: additional logo asset

Other
-----
- logo.svg
  A copy of the logo SVG at the root of the ui folder (separate from assets/).
